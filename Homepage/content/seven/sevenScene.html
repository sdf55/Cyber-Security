Attacker is Added to Code Deploy Group

If the attacker has access to the code, then they could insert malicious code before deployment. They will then have access to all the accounts that use the program with the inserted malicious code.

Want to prevent one person to having access to both develop and deploy anything themselves. This raises the importance of exercising the separation of duties. Separation of duties is the act of requiring multiple people to complete a task. Not allowing any one user too much control can help prevent attacks on AWS IAM users.
