Discovering when an AWS role or group is alterred

It is important to keep track of all ongoing activity logs of users to make sure that they are only using the applications that they have been given permission to.

AWS has logging features that can determine what actions users are taking as well as the resources they are using.

What Do Log Files Show?

The log files show:

  • Time and date of actions
  • Source IP for an action
  • Actions that failed due to inadequate permissions and more

Many of AWS’s services have logging capabilities:

Amazon CloudFront

“Logs user requests that CloudFront receives.” More Information: https://aws.amazon.com/cloudfront/

AWS CloudTrail

“Logs AWS API calls and related events made by or on behalf of an AWS account.” More Information: https://aws.amazon.com/cloudtrail/

Amazon CloudWatch

“Monitors your AWS cloud resources and the applications you run on AWS. You can set alarms in CloudWatch based on metrics that you define.” More Information: https://aws.amazon.com/cloudwatch/

AWS Config

“Provides detailed historical information about the configuration of your AWS resources, including your IAM users, groups, roles, and policies. AWS Config can determine the permissions that belonged to a user or group at a specific time.” More Information: https://aws.amazon.com/config/

Amazon Simple Storage Service (Amazon S3)

“Logs access requests to your Amazon S3 buckets.” More Information: https://aws.amazon.com/s3/
